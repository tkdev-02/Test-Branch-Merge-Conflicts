 - Branching and Navigation
git branch 			â†’ Lists all local branches
git branch <branch-name> 	â†’ Creates a new branch
git checkout <branch-name> 	â†’ Switches to another branch
git checkout -b <branch-name> 	â†’ Creates and switches to a new branch
git branch -d <branch-name> 	â†’ Deletes a branch

 - Committing Changes
git add . 			â†’ Adds all changes to staging
git add <file> 			â†’ Adds a specific file to staging
git commit -m "Thandaza"	â†’ Commits changes with a message

 - Pushing and Pulling
git push origin <branch-name> 		â†’ Pushes branch to remote
git push -u origin <branch-name> 	â†’ Pushes branch and sets upstream
git pull origin <branch-name> 		â†’ Fetches and merges changes from remote
git fetch 				â†’ Fetches latest changes but does not merge

 - Checking History
git log 				â†’ Shows commit history
git diff 				â†’ Shows changes not staged for commit
git diff --staged 			â†’ Shows changes staged for commit

 - Undo & Reset Changes
git reset --soft HEAD~1 		â†’ Undo last commit, keep changes staged
git reset --mixed HEAD~1 		â†’ Undo last commit, keep changes unstaged
git reset --hard HEAD~1 		â†’ Undo last commit and discard changes
git checkout -- <file> 			â†’ Discards changes in a file
git revert <commit-hash> 		â†’ Creates a new commit that undoes the changes of a specific commit

 - Working with Remote Repositories
git remote -v 				â†’ Shows remote repository URLs
git remote add origin <repo-url> 	â†’ Links local repo to remote
git remote remove origin 		â†’ Removes remote repository link

 - Stashing Changes (Temporary Save)
git stash 			â†’ Saves uncommitted changes temporarily
git stash pop 			â†’ Applies last stashed changes
git stash list 			â†’ Shows stashed changes
git stash drop 			â†’ Deletes a stash

 - Tagging Commits
git tag <tag-name> 			â†’ Creates a tag
git tag -a <tag-name> -m "Thandaza" 	â†’ Creates an annotated tag
git tag 				â†’ Lists all tags
git push origin <tag-name> 		â†’ Pushes a tag to remote

 - Bonus Commands
git show <commit-hash> 		â†’ Shows details of a specific commit
git shortlog 			â†’ Summarizes commit history per author
git blame <file> 		â†’ Shows who last modified each line of a file

Typical workflow:

1. Clone the repositry
2. Create a new branch from the *MAIN* or another branch
3. Make your changes
4. Push the branch to the remote repo
5. Open a Pull Request
6. Merge the changes
7. Pull the merged changes into your local *MAIN* branch
8. Repeat from Step 2

How to resolve Merge Conflicts

1. Checkout Main branch
 - git checkout main

2. pull latest changes from Main branch
 - git pull
 - Now local main branch and remote main branch are ==

3. Now checkout to your branch
 - git checkout branch-name

4. 
 - git merge main

 - Now merged MAIN branch into YOUR branch
 - initial aim was to merge your branch into the main
 - but due to merge conflict, it didnt work
 - so we must first resolve the issue
 - -> merge the main into our branch to Identify the problem

 - 'git merge branch-name', merges branch-name into the branch your currenlty in(last checked into)


5. Code editor 

6. 
 - git add .
 - git commit -m 'resolve merge conflicts'
 - git push
